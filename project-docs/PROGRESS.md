# Rental Buddy 進度總表

最後更新：2026-05-06

## 目前產品結構

- 頂層頁面：`查核表`、`報告`
- 報告內檢視模式：
  - `手機版檢視`（友善閱讀）
  - `精簡版`（正式文件預覽）
- 匯出規則：`匯出文件（PDF）` 只輸出精簡版內容

## 已完成項目

### 1) 核心資料與狀態
- 看房紀錄支援多筆，使用 `localStorage` 持久化。
- 每筆紀錄支援地址、月租金、各項 checklist 狀態與備註。
- 評分邏輯統一為 100 分制，並提供 A/B/C 評等。
- 評等門檻已調整為較嚴格版本：
  - A：85 以上
  - B：65-84
  - C：64 以下

### 2) 查核表體驗
- 新增月租金輸入欄位（與地址並列於基本資料區）。
- 紀錄切換、新增、刪除流程可用。
- 查核項可勾選/標記問題/展開補備註。

### 3) 報告頁體驗
- 改為「房屋評價」導向，不再使用性格測驗語氣。
- 報告頁可切換手機版檢視與精簡版預覽。
- 手機版維持較溫馨、可快速掃描的資訊架構。

### 4) 精簡版與匯出
- 精簡版採檢驗單風格（分欄、表格、雷達、重點備註）。
- 匯出流程改為列印文件（非截圖），並鎖定輸出精簡版內容。
- 精簡版在手機寬度的跑版問題已修正：
  - 單欄化
  - 雷達縮放
  - 表格可讀性調整
  - 操作按鈕避免擠壓

### 5) 桌機版版面穩定性
- 調整桌機 header/footer/content 最大寬度與對齊。
- 改善捲動時覆蓋與視覺錯位問題。

### 6) PWA 與部署
- 已加入 PWA（manifest、icons、service worker）。
- 已修正 GitHub Pages 空白頁問題（`baseHref` 使用相對路徑）。
- 已同步 `docs/`，Pages 可部署最新版本。

## 開發與部署注意

- 開發指令：`npm start`（已改為 polling 監看，降低 `EMFILE` 影響）。
- 正式建置：`npm run build`
- GitHub Pages：目前使用 `docs/` 產物部署，推送時需要同步 `docs/`。

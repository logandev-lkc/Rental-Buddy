# Rental Buddy 下一階段開發文件

最後更新：2026-05-06（夜間規劃版）

## 目標

這份文件用來管理「接下來要加的功能」，讓需求、切片、驗收標準與發布紀錄集中在同一處。

---

## 開發原則

1. 先定義使用情境，再定義 UI 與資料欄位。
2. 每個功能都要有「可驗收標準」。
3. 每次實作盡量小步快跑（1 個功能 = 1 個可演示版本）。
4. 若涉及 GitHub Pages，推送前需同步 `docs/`。
5. 若功能牽涉第三方服務（地圖、登入、金流），先做最小可行串接（MVP）再擴充。

## 建議切片路線圖

### Phase A：查核表體驗先變強（高優先）
- F-002 查核表版面重構

預期成果：

- F-001 Google 地圖整合

- F-009 Buy me a coffee

預期成果：
- 提升實用性與延展性。

## 功能細節草案（待逐項討論定稿）

### F-001 Google 地圖整合（地址定位）
- **MVP 目標**：從查核表與報告頁可一鍵開啟 Google Maps，確認地址與周邊。
- **細節草案**：
  - 在查核表基本資料區新增「開啟地圖」按鈕（第一版唯一入口）。
  - 先採外連 Google Maps（免 API Key），優先快速上線。
  - 地址可不完整，仍允許保存與開圖，不強制完整地址。
- **本輪定稿**：
  - 入口位置：查核表（`checklist_only`）。
  - 資料規則：可保存不完整地址（`allow_fuzzy + keep_incomplete`）。
  - 範圍策略：先外連，之後再評估「地圖選點回填」進階版。

### F-002 查核表版面重構（手機優先）
- **MVP 目標**：在看房現場更快勾選，不需要頻繁捲動或找按鈕。
- **細節草案**：
  - 上方摘要列固定：目前紀錄、地址、月租金、完成度。
  - 分類切換改成下拉選單（省空間，提升同屏資訊量）。
  - 底部動作列固定：立即儲存 / 查看報告 / 新增紀錄。
- **本輪定稿**：
  - 版面優先：同屏資訊量優先（`more_info_visible`）。
  - 導覽方式：分類下拉（`dropdown`）。
  - 固定按鈕：`save_now`、`goto_report`、`new_record`。

### F-003 新增「氣味」檢查項
- **MVP 目標**：加入實際租屋中常見卻易忽略的嗅覺問題。
- **細節草案**：
  - 新增項目：
    - 是否有菸味（公共區域/室內）
    - 是否有煮菜油煙味（鄰戶或管道）
  - 加入提示文案（建議在哪些時段觀察更準）。
  - 納入報告與精簡版統計。
- **本輪定稿**：
  - 所屬分類：`環境鄰居`（`env_neighbor`）。
  - 輸入方式：三級（無異味 / 可接受 / 明顯異味）。
  - 顯示策略：保留「建議觀察時段」短提示（`need_hint`）。

### F-004 細節改為選單式輸入
- **MVP 目標**：降低文字輸入負擔，提升填寫率。
- **細節草案**：
  - 常用狀態改成 chips：
    - 良好 / 普通 / 偏差
    - 有 / 無 / 不確定
  - 保留「其他備註」自由輸入欄。
  - 選單值同步進報告細節。
- **本輪定稿**：
  - 第一批覆蓋：安全、設備、費用、環境（全高頻分類）。
  - 互動元件：單選 chips（`single_select_chip`）。
  - 備註策略：每個選單項都可補充備註（`keep_note_all`）。

### F-005 報告頁 UX 優化
- **MVP 目標**：手機上更快得到「是否值得租」判斷。
- **細節草案**：
  - 報告開頭固定顯示總評、分數、風險等級。
  - 詳細區塊預設收合，可展開看細項。
  - 比較與雷達區塊改為次要資訊，避免干擾第一判讀。
- **本輪定稿**：
  - 比較房源：獨立比較頁（`separate_page`）。
  - 細項預設：收合（`collapsed`）。
  - 風險訊號：強提示（色塊 + icon + 標籤，`strong`）。

### F-006 精簡版（文件版）優化
- **MVP 目標**：精簡版可直接交付，且匯出結果一致。
- **細節草案**：
  - 桌機版加強字級層次與欄寬比例。
  - 手機版預覽維持單欄，不跑版。
  - 強化評分標準說明（A/B/C 定義固定顯示）。
- **本輪定稿**：
  - 文件欄位：先不加簽名區（`no_signature`）。
  - 版面密度：平衡型（`balanced`）。
  - 手機預覽：盡量接近列印版（`near_print_match`）。

### F-007 上傳資料（附件）
- **MVP 目標**：每筆記錄可附現場照片作為佐證。
- **細節草案**：
  - 每筆記錄可上傳 1~5 張圖片。
  - 先存本機（localStorage / IndexedDB，需評估容量）。
  - 報告與精簡版顯示附件數量（必要時顯示縮圖）。
- **本輪定稿**：
  - 儲存方式：IndexedDB（存圖片檔案）。
  - 顯示方式：小縮圖（最多顯示 3 張）+ 附件總數。
  - 上傳上限：每筆最多 10 張（`limit_10`）。

### F-008 登入系統（待決策）
- **MVP 目標**：跨裝置同步（若確認需要）。
- **細節草案**：
  - 先不啟動，保留設計位置。
  - 若要做，建議從 OAuth（Google）或 magic link 開始。
  - 需定義資料隱私與匯出刪除流程。
- **本輪定稿**：
  - 啟動時機：Phase B 後啟動（`after_phase_b`）。
  - 登入方式：Google OAuth 優先（`google_oauth`）。
  - 同步策略：完整同步（含紀錄與附件，`full_sync`）。

### F-009 Buy me a coffee
- **MVP 目標**：提供支持作者入口，不打擾主流程。
- **細節草案**：
  - 入口放在設定頁或關於頁，不放在查核主動線。
  - 外連到贊助頁（Buy Me a Coffee）。
  - 保留簡單 CTA 文案，不做付款流程嵌入。
- **本輪定稿**：
  - 入口位置：頁尾固定連結（`footer_link`）。
  - 文案語氣：中性（`neutral`）。
  - 顯示策略：固定顯示（`always_visible`）。

### F-010 PWA 安裝引導提示
- **MVP 目標**：讓使用者知道可安裝成 App，不漏掉既有能力。
- **細節草案**：
  - 監聽 `beforeinstallprompt`，顯示安裝提示卡。
  - 提供「稍後提醒」與「不再顯示（本機）」。
  - iOS 補充手動加入主畫面圖文引導。
- **本輪定稿**：
  - 觸發時機：完成第一筆紀錄後提示（`after_first_record`）。
  - 提示內容：Android 用原生安裝 prompt，iOS 顯示圖文引導。
  - 使用者控制：保留稍後提醒與不再顯示。

### F-011 離線狀態提示條
- **MVP 目標**：離線時給明確狀態，避免誤判資料是否遺失。
- **細節草案**：
  - 以頂部狀態條顯示「離線中 / 已恢復連線」。
  - 離線仍可編輯，並提示資料保存在本機。
  - 連線恢復後短暫顯示成功提示再自動收合。
- **本輪定稿**：
  - 離線提示：顯示重新整理按鈕（`with_reload`）。
  - 恢復提示：顯示 4 秒後自動收合（`4s`）。
  - 資料保護：固定顯示「資料已保存在本機」文案。

### D-005：報告閱讀與列印預覽分離

- **決議日**：2026-05-16
- **背景**：現有「圖文版／表格版」易被理解為兩種內容；表格版實為 `print-sheet` 預覽，與 v2「螢幕 ≠ PDF」原則一致但入口混淆。
- **選項**：（A）維持頂部雙切換；（B）預設閱讀 + 匯出前全螢幕列印預覽；（C）只保留一種畫面並直接列印。
- **結果**：採用 **方案 B**。
- **影響範圍**：`report-toolbar`、`reportViewMode`、`app.html` 報告區、`F-023`。
- **相關**：[specs/report-v4-spec.md](./specs/report-v4-spec.md) §2、[feature-roadmap.md](./feature-roadmap.md)

---

### D-006：PDF 技術路線

- **決議日**：2026-05-16
- **背景**：需單頁 InBody PDF；專案為本機 PWA、無後端；`package.json` 含 `html2canvas` 但未用於匯出。
- **選項**：（A）`window.print()`；（B）html2canvas + jsPDF；（C）pdfmake／pdf-lib；（D）伺服器 Puppeteer。
- **結果**：v4 採用 **方案 A**；若實機仍破頁，先加內容預算與 `scale()` 保險，再評估（C）為 **v5 候選**，（D）違反 **D-001** 產品定位。
- **影響範圍**：`exportReportAsDocument()`、`app.css` `@media print`、`F-020`。
- **相關**：v2 [inbody-report-gen-spec.md](../v2/inbody-report-gen-spec.md) §6

---

### D-007：多房源比較降級

- **決議日**：2026-05-16
- **背景**：比較區（pills、多曲線雷達）拉長報告、分散決策首屏注意力。
- **選項**：（A）維持首屏；（B）收合區塊；（C）移至獨立「比較」流程。
- **結果**：採用 **方案 B** 為 v4 必達；**方案 C** 可作後續增強。
- **影響範圍**：報告 HTML 區塊順序、`compare-pills`、`F-024`。
- **相關**：[specs/report-v4-spec.md](./specs/report-v4-spec.md) §3.4

---

### D-008：列印版附件策略

- **決議日**：2026-05-16
- **背景**：3 張附件縮圖 + 長 meta 為 PDF 超頁主因之一。
- **選項**：（A）0 張，僅文字；（B）1 張小圖 + 共 N 張說明；（C）維持 3 張。
- **結果**：v4 採用 **方案 B**（實作驗收若仍超頁可改 **方案 A** 並更新本條）。
- **影響範圍**：`print-card--attachments`、`F-020`。
- **相關**：[specs/report-v4-spec.md](./specs/report-v4-spec.md) §4.3
